# Provenance Guard — Planning / Spec

> Written before implementation. This is both the design record and the prompt
> source used to drive AI code generation in Milestones 3–5.

## Problem

Creative platforms need to tell readers whether posted text was written by a
human or generated by AI — **to protect attribution and trust, not to police
creativity**. The system must classify text, honestly express uncertainty,
surface a plain-language label, accept appeals, and keep an audit trail. It must
be conservative: on a writing platform, **falsely branding a human's work as AI
is the worst outcome**.

## Architecture narrative (path of one submission)

A creator's text arrives at `POST /submit` with `text` + `creator_id`. Input is
validated at the boundary. The text is fanned out to **two independent signals**:
the **LLM semantic classifier** (Groq) and the **stylometric heuristics** module.
Each returns a probability that the text is AI-generated, `P(AI) ∈ [0,1]`. The
**scorer** combines them into a single `p_ai`, derives an **attribution**
(`likely_ai` / `uncertain` / `likely_human`) using asymmetric thresholds, and a
**confidence** value. The **label builder** turns that into reader-facing text.
The result is written to the **submissions** table and an entry is appended to
the **audit_log**. The response returns `content_id`, attribution, confidence,
per-signal detail, and the label.

For an appeal, `POST /appeal` takes the `content_id` + `creator_reasoning`, sets
the submission's status to `under_review`, appends an `appeal` event (carrying
the original decision *and* the creator's reasoning) to the audit log, and
confirms receipt. No automated re-classification — a human reviewer owns it.

## Architecture

```
                          ┌──────────────────────────────────────────────┐
 SUBMISSION FLOW          │                 PROVENANCE GUARD             │
                          │                                              │
 client                  POST /submit                                    │

   │                                          │ raw text                 │
   │                          ┌───────────────┴───────────────┐          │
   │                          ▼                               ▼          │
   │              Signal 1: LLM (Groq)          Signal 2: Stylometry     │
   │              semantic / "voice"            burstiness, TTR, punct   │
   │                 │ P(AI)=llm_score             │ P(AI)=stylo_score   │

   │                    Scorer: p_ai = 0.65·llm + 0.35·stylo             │

   │                                 ▼                                    │
   │                         Label builder → label text                  │
   │                                 │                                    │
   │                                 ├──────────► submissions table       │
   │                                 ├──────────► audit_log (classify)    │
   │  {content_id, attribution,      ▼                                    │
   │◄─ confidence, signals, label} response                              │
                          │                                              │
 APPEAL FLOW              │                                              │
   │ {content_id,        POST /appeal                                     │
   │  creator_reasoning} ──────────►  lookup content_id                   │
   │                                     │ set status = under_review      │
   │                                     ├──────────► submissions (update)│
   │                                     ├──────────► audit_log (appeal)  │
   │◄─ {status: under_review} ───────────┘  + original decision           │
                          │                                              │
   GET /log  ────────────►  read recent audit_log entries ──► JSON       │
                          └──────────────────────────────────────────────┘
```

**Submission flow:** raw text → two signals (each `P(AI)`) → weighted scorer →
attribution + confidence → label → persisted to `submissions` and `audit_log` →
JSON response. **Appeal flow:** `content_id` → status flips to `under_review` →
appeal event (with original decision + creator reasoning) appended to
`audit_log` → confirmation.

## The five required questions

### 1. Detection signals

| # | Signal | Type | Measures | Output |
|---|--------|------|----------|--------|
| 1 | **Groq LLM** (`llama-3.3-70b-versatile`) | Semantic | Holistic coherence, originality of phrasing, idiosyncrasy / "voice" | `P(AI) ∈ [0,1]` + rationale |
| 2 | **Stylometric heuristics** | Structural | Surface statistics: sentence-length burstiness, vocabulary diversity, punctuation | `P(AI) ∈ [0,1]` + raw metrics |

**Why these two:** they are genuinely independent — one reads *meaning*, the
other reads *form*. A signal that fools one (e.g. AI text deliberately given
varied sentence lengths) often won't fool the other. Combining a semantic and a
structural view is more informative than either alone.

**Stylometry sub-metrics:**
- **Burstiness** = coefficient of variation of sentence length. Human prose is
  bursty (long + short sentences); AI prose is uniform. *Primary weight (0.55).*
- **Type-token ratio (TTR)** = unique/total words (first 200). Mid-range TTR is
  AI-typical; very low (repetitive/poetic) or very high (terse/casual) skews
  human. Scored with a gaussian centered at 0.5. *Weight 0.25.*
- **Expressive punctuation density** (`! ? ... --`). Sparse/even → AI; bursty →
  human. *Weight 0.20.*
- **Casual-marker discount** (`i`, `ok`, `gonna`, `honestly`, `lol`…): pulls the
  score toward human, protecting informal writers from false AI flags.

**Combining → single confidence:** `p_ai = 0.65·llm_score + 0.35·stylo_score`
(LLM weighted higher; if the LLM is unavailable, stylometry takes full weight).
`confidence = max(p_ai, 1-p_ai)` ∈ [0.5, 1.0].

**Blind spots (every signal has them):**
- *LLM:* can be wrong on short text, non-English, or domain jargon; non-deterministic-ish; costs a network call. Can be gamed by paraphrasing.
- *Stylometry:* short text has no sentence variance to measure (CV=0 → looks AI). Formal human writing (legal, academic) is uniform → false AI. Poetry with heavy repetition → false AI. It measures form, never truth.

### 2. Uncertainty representation

- `p_ai` is the system's probability the text is AI-generated.
- **Asymmetric thresholds** (false-positive aversion):
  - `p_ai ≥ 0.70` → **likely_ai** (high bar — we don't accuse lightly)
  - `p_ai ≤ 0.40` → **likely_human**
  - otherwise → **uncertain** (reported honestly, never forced to a side)
- `confidence = max(p_ai, 1-p_ai)`. A `p_ai` of 0.51 → confidence 0.51 →
  *uncertain* label; a `p_ai` of 0.95 → confidence 0.95 → *high-confidence AI*
  label. **Different scores produce different labels — no binary flip at 0.5.**
- **What 0.6 means:** "leaning AI but not sure" → lands in the uncertain band,
  and the label says so plainly rather than declaring a verdict.

### 3. Transparency label design (three variants — exact text)

**High-confidence AI** (`likely_ai`):
> 🤖 Likely AI-generated — Our analysis suggests this content was probably created with the help of AI ({pct}% confidence). This is an automated estimate, not a certainty — AI detection is imperfect. If you wrote this yourself, you can appeal.

**High-confidence human** (`likely_human`):
> ✍️ Likely human-written — Our analysis suggests this content was probably written by a person ({pct}% confidence). This is an automated estimate and is not a guarantee.

**Uncertain** (`uncertain`):
> ❓ Uncertain origin — We couldn't confidently determine whether this content was written by a person or generated by AI (only {pct}% certainty either way). We're showing this honestly rather than guessing. Treat the origin as unknown.

`{pct}` = `round(confidence × 100)`, injected at render time.

### 4. Appeals workflow

- **Who:** the creator of the content (identified by `content_id` from their
  submission response).
- **They provide:** `content_id` + `creator_reasoning` (free text).
- **System does:** look up the submission → set `status = under_review` →
  append an `appeal` audit event carrying *both* the original decision (so the
  reviewer sees what was contested) and the creator's reasoning → return
  confirmation. **No automated re-classification.**
- **A human reviewer opening the queue sees:** the original text, the original
  attribution + confidence + both signal scores, and the creator's stated
  reasoning — everything needed to make a call. (Surfaced today via `GET /log`,
  filtered to `event_type = appeal`.)

### 5. Anticipated edge cases (specific)

1. **Very short submissions** (one sentence / a tweet). Stylometry has no
   sentence-length variance to measure (CV = 0), so burstiness pegs the text as
   "uniform" → false AI lean. *Mitigation:* LLM weight dominates; documented as a
   known limitation; appeal path exists.
2. **Formal human writing** (academic, legal, financial prose). Naturally
   uniform sentence length and mid-range TTR → stylometry leans AI. This is
   exactly why the AI threshold is high (0.70) and the casual discount exists.
3. **Lightly-edited AI output.** A human polishing AI text sits genuinely
   between classes — the correct output is *uncertain*, which the three-band
   design produces rather than a false-confident verdict.
4. **Repetitive/structured poetry.** Low TTR + regular line lengths can read as
   AI; the LLM signal and the appeal path are the backstops.

## AI Tool Plan

**M3 — submission endpoint + first signal**
- *Provide:* Detection-signals section + Architecture diagram.
- *Ask for:* Flask skeleton with `POST /submit` stub returning a hardcoded
  response; the LLM `llm_signal(text)` function returning `{available, score, rationale}`.
- *Verify:* call `llm_signal` directly on 2–3 inputs; confirm it returns a float
  in [0,1] and degrades cleanly with no key; confirm route accepts JSON and
  echoes a `content_id`.

**M4 — second signal + confidence scoring**
- *Provide:* Detection-signals + Uncertainty-representation + diagram.
- *Ask for:* `stylo_signal(text)` (3 metrics + casual discount) and `combine()`.
- *Check:* generated thresholds match the spec (0.70 / 0.40); scores diverge
  meaningfully between clearly-AI and clearly-human text; print both signal
  scores separately to find a misbehaving one. (Captured in `tests/test_pipeline.py`.)

**M5 — production layer**
- *Provide:* Label-variants + Appeals-workflow + diagram.
- *Ask for:* `build_label(result)` mapping confidence → the correct variant; the
  `POST /appeal` endpoint.
- *Verify:* all three variants reachable by feeding different confidences; an
  appeal flips status to `under_review` and writes an audit row with the
  reasoning; rate-limit returns 429 past the cap.
